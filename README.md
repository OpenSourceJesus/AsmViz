# Assembly Visualizer

A PyQt5-based application for visualizing call graphs and assembly code of C source files. The tool extracts function call relationships using pycparser and displays the corresponding assembly code generated by GCC, along with register usage analysis.

## Features

- **Call Graph Extraction**: Parse C source files and extract function definitions and call relationships
- **Assembly Code Display**: Automatically extract and display assembly code for each function using GCC
- **Register Analysis**: 
  - Shows registers used within each function
  - Shows registers used across the entire call graph (function + all callees)
  - Color-coded register highlighting in assembly code
- **Interactive Visualization**:
  - Pan the view with left, right, or middle mouse button
  - Zoom with mouse wheel
  - Drag nodes to manually reposition them
  - Automatic circular layout option
- **Visual Call Relationships**: Arrows connect function calls, pointing from the call site in the caller's assembly to the entry point of the callee
- **Color-Coded Assembly**: Instructions and registers are highlighted with different colors for better readability

## Requirements

- Python 3.6+
- Python packages: `pycparser`, `PyQt5`
- System tools: `gcc` (for assembly extraction) and C preprocessor (`cpp`)

## Installation

1. Install Python dependencies:
```bash
pip install -r requirements.txt
```

2. Ensure GCC is installed:
   - **Linux**: Usually pre-installed or available via package manager (`sudo apt-get install gcc` on Debian/Ubuntu)
   - **macOS**: Install Xcode Command Line Tools (`xcode-select --install`)
   - **Windows**: Install MinGW or use WSL

Note: `pycparser` requires a C preprocessor. On Linux, you typically need `gcc` or `clang` installed. On Windows, you may need to install MinGW or use the `cpp` argument.

## Usage

Run the application with a C file:
```bash
python main.py example.c
```

Or run without arguments and use the GUI:
```bash
python main.py
```

Or directly:
```bash
python call_graph_visualizer.py [filename.c]
```

### Using the Application

1. **Command-line**: Pass a C file as an argument to automatically load and visualize it
   ```bash
   python main.py your_file.c
   ```

2. **GUI**: Click "Open C File" to select a C source file
3. The call graph and assembly code will be automatically extracted and displayed
4. **Navigation**:
   - **Pan**: Click and drag with left, right, or middle mouse button
   - **Zoom**: Use mouse wheel to zoom in/out

## Visual Elements

The visualizer displays each function as a detailed node containing:

- **Function Signature**: The function's declaration at the top
- **Register Usage (Call Graph)**: All registers used in the function and its entire call graph
- **Register Usage (Function Only)**: Registers used only within this specific function
- **Assembly Code**: The complete assembly code for the function, with:
  - Color-coded instructions
  - Color-coded registers (related registers like `rax`, `eax`, `ax`, `al` share the same color)
  - Clean formatting with labels and directives preserved

**Call Relationships**: Arrows connect functions, showing:
- The exact line in the caller's assembly where the function is called
- The entry point of the callee function
- Visual flow from caller to callee

## How It Works

1. **call_graph_extractor.py**: Uses pycparser to parse C files and extract:
   - Function definitions
   - Function calls within each function
   - Builds a call graph data structure

2. **assembly_extractor.py**: Extracts assembly code by:
   - Running `gcc -S` to generate assembly from the C source
   - Parsing the assembly file to extract individual function bodies
   - Extracting function signatures from the AST

3. **call_graph_visualizer.py**: PyQt5 GUI that:
   - Loads C files and extracts call graphs
   - Extracts assembly code and register usage
   - Displays functions as detailed nodes with assembly code
   - Provides interactive visualization with pan, zoom, and layout features
   - Draws call relationships as arrows connecting call sites to entry points

## Project Structure

- `main.py`: Entry point for the application
- `call_graph_extractor.py`: Extracts function definitions and call relationships from C files
- `assembly_extractor.py`: Extracts assembly code and function signatures
- `call_graph_visualizer.py`: Main GUI application with visualization features
- `Tests/`: Contains example C files for testing

## Limitations

- Only processes single C files (not multi-file projects)
- Requires C preprocessor for pycparser
- Requires GCC for assembly extraction
- Function pointer calls may not be fully resolved
- Recursive calls are shown but may create visual clutter
- Assembly extraction depends on GCC's output format

## Troubleshooting

If you encounter parsing errors:
- Ensure your C file is valid C code
- Make sure a C preprocessor (`cpp`) is available
- Check that all necessary headers are accessible
- Verify that GCC is installed and accessible in your PATH

If assembly extraction fails:
- Ensure GCC is installed and working (`gcc --version`)
- Check that the C file compiles successfully
- Verify file permissions and paths
